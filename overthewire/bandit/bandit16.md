## Summary

This challenge is the level16 for the Bandit series on OverTheWire, focusing on basic Linux command-line skills. The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL/TLS and which donâ€™t. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it..
---

## Approach

Since this is an basic challenge, I planned to use basic Linux commands like `ssh`, `openssl`, `grep` and `ps` to explore the server and retrieve the flag.

---

## Solution Walkthrough

Connecting to the server using the `ssh` command as below:

```bash
ssh bandit16@bandit.labs.overthewire.org -p 2220
# Password: kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx

```

After logging in, First we need to find the port in which its running. We know the range of the port is from 31000 to 32000. We can use the `ps` command to list all the process running on the particular port as shown below.  

```bash
ps -ef | grep 31
# Output:
# bandit16 3356203 3323072  0 14:53 pts/112  00:00:00 ncat --ssl localhost 31790
# bandit16 3477879 3477629  0 15:16 pts/63   00:00:00 nmap -A -p 31000-32000 localhost
# bandit16 3483324 3477629  0 15:21 pts/63   00:00:00 openssl s_client -connect localhost:31790
# bandit16 3487714 3479439  0 15:26 pts/44   00:00:00 openssl s_client -connect localhost:31790
# bandit16 3491154 3490953  0 15:29 pts/76   00:00:00 grep --color=auto 31
```

Now It clearly shows the the there is a process that is running in port `31790` and it is an `open ssl` client. we can use the `openssl` command as shown above in the output of the previous command. Note that we need to use `-ign_eof` inorder to avoid the `KEYUPDATE` bug in the challange.  The output here is an RSA private key which needs to be stored in file so that we can use to login to the next challange

## Lessons Learned

It was a easier challange, the first trick was to find the port in which the applicaiton running. After finding that i was stuck with the KEYUPDATE error whcih didn't let me submit the password to the process. I had to search and find the we need to use `ign_eof` flag in the cli to prevent the error.
